How to edit the database

Option 1 (preferred): Editting entries using the GUI

When you want to edit the status of competencies, e.g. because you believe a researcher in actually having a specific competency, or if a researcher doesn't want to be associated with another competency, you can accomplish this by logging in as an admin (done, otherwise you're [hopefully] not able to read this) and navigating to the competency of interest using the searchbar.
The color of the navbar and status buttons indicate that you're in admin mode.

Admin View (logged in) User view (not logged in)

Now you can edit the visibility of competencies (only for specific authors). The check mark means verified, the open eye unvalidated and the crossed eye hidden.

After some editing of the admin view and refreshing of the user view, you can see, that Sascha Wirges entry is hidden, Adrian Englhardts entry is verified and Finn Sartoris entry remains untouched.

Option 2: Use a database viewer

Open the database.db (located in database_api) with a database viewer like DBeaver and edit the database manually. But please make a backup before and edit at your own risk.
